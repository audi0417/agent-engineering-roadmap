# Enterprise Agent Operating Checklist

Use this checklist when a team wants to launch or review an internal agent.

## Registry

- [ ] Agent name
- [ ] Owner
- [ ] Business purpose
- [ ] Risk tier
- [ ] Data sources
- [ ] Tools
- [ ] Eval suite
- [ ] Incident owner

## Pre-Launch Review

- [ ] Tool scopes follow least privilege
- [ ] Memory retention is documented
- [ ] Prompt injection risks are evaluated
- [ ] Approval gates exist for high-risk actions
- [ ] Observability traces are enabled
- [ ] Cost and latency budgets are defined
- [ ] Rollback or disable path is tested

## Ongoing Review

- [ ] Access review cadence is defined
- [ ] Eval results are monitored
- [ ] Incidents become regression cases
- [ ] Inactive agents are retired
- [ ] Owners confirm the agent is still needed
