# Good First Issues

These issues are intentionally small, concrete, and friendly to first-time contributors. They help the project grow beyond a static course into a community-maintained learning roadmap.

## Resolved Starter Tasks

The first starter issues were resolved in the 2026-06-30 teaching refresh. They remain useful as examples of small, well-scoped contribution ideas.

1. [Add sample learner submissions for Lab 00 and Lab 01](https://github.com/audi0417/agent-engineering-roadmap/issues/1)
   - Good for learners who want to show what a strong answer looks like before writing code.
   - Adds examples for agent specs, failure-mode thinking, and basic tool design.

2. [Add two more cases to the Healthcare Agent Casebook](https://github.com/audi0417/agent-engineering-roadmap/issues/2)
   - Good for contributors interested in high-stakes AI safety.
   - Expands the casebook with realistic triage, refusal, escalation, and audit scenarios.

3. [Add path-level eval cases to the Graph Approval Agent](https://github.com/audi0417/agent-engineering-roadmap/issues/3)
   - Good for contributors who want to practice testing agent workflows.
   - Improves coverage for approval gates, escalation paths, and graph transition behavior.

4. [Create a visual quickstart section for the README](https://github.com/audi0417/agent-engineering-roadmap/issues/4)
   - Good for contributors with documentation or design instincts.
   - Makes the repo easier to understand in the first 30 seconds.

5. [Add finance no-advice eval cases to the capstone starter](https://github.com/audi0417/agent-engineering-roadmap/issues/5)
   - Good for contributors interested in financial-agent guardrails.
   - Adds tests that distinguish educational finance support from personalized investment advice.

## Contributor Guidance

Future first issues should stay narrow:

- Change one folder or learning artifact where possible.
- Add or update a short README section when behavior changes.
- Run `python scripts/verify_examples.py` before opening a pull request when examples are affected.
- Prefer realistic examples over generic placeholder text.
