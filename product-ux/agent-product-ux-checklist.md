# Agent Product UX Checklist

Use this checklist before shipping an agent to users.

## Visibility

- [ ] Users can see the current agent step
- [ ] Tool calls are visible before high-risk execution
- [ ] Retrieved evidence can be inspected
- [ ] The system separates facts, assumptions, and suggestions

## Control

- [ ] Users can stop the agent
- [ ] Users can approve or reject high-risk actions
- [ ] Users can escalate to a human
- [ ] Undo or rollback is available where possible

## Trust

- [ ] Refusals explain the safe alternative
- [ ] The agent states uncertainty when needed
- [ ] The agent does not hide failed tool calls
- [ ] Audit history is available for important actions

## Recovery

- [ ] Failed runs can be retried safely
- [ ] Users know what changed and what did not
- [ ] Incident links or support paths are visible for high-risk workflows
